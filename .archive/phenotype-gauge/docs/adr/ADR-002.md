# ADR-002: Client Library Design

## Status
**Accepted**

## Context
Metrics libraries must be idiomatic for each language while maintaining consistent concepts.

## Decision
- Rust: prometheus crate integration
- Go: prometheus/client_golang style
- Python: prometheus_client compatibility
- TypeScript: prom-client inspired API
- Common metric types across languages
- Push and pull model support

## Consequences
- Idiomatic APIs per language
- Consistent metric behavior
- Maintenance burden across languages
