# ADR-003: Metric Naming and Labeling

## Status
**Accepted**

## Context
Consistent naming enables cross-service metric aggregation and dashboard reusability.

## Decision
- Namespace: `phenotype_<service>_<metric>`
- snake_case naming
- Standard labels: service, instance, version
- Custom labels for business dimensions
- Label cardinality limits
- Documentation requirements

## Consequences
- Query consistency
- Dashboard reusability
- Enforced naming discipline
