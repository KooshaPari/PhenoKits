# ADR-001: Python Middleware Architecture

## Status
**Accepted**

## Context
Python applications in the Phenotype ecosystem need consistent middleware patterns for cross-cutting concerns.

## Decision
- ASGI/WSGI dual compatibility
- Starlette-based implementation
- Middleware stack ordering
- Context propagation
- Async-first design
- Framework-agnostic core

## Consequences
- Works with FastAPI, Flask, Django
- Modern async Python patterns
- Some complexity for sync frameworks
