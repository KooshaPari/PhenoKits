# ADR-005: Error Handling Middleware

## Status
**Accepted**

## Context
Consistent error responses improve API usability and client error handling.

## Decision
- RFC 7807 Problem Details format
- Exception to HTTP status mapping
- Stack trace in development
- Generic messages in production
- Error ID for support
- Structured error logging

## Consequences
- Consistent error format
- Safe error exposure
- Debugging support

## References
1. [RFC 7807](https://tools.ietf.org/html/rfc7807)
