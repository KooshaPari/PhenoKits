# ADR-003: Type Evolution and Compatibility

## Status
**Accepted**

## Context
Types evolve; backward and forward compatibility must be maintained.

## Decision
- Field number reservation
- Optional fields for evolution
- No required fields after initial release
- Wire format compatibility testing
- Breaking change detection in CI
- Deprecation annotations

## Consequences
- Safe type evolution
- Some design constraints
- Testing overhead
