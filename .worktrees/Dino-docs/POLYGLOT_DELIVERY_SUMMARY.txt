BOTTLENECK ANALYSIS:

Asset Pipeline Bottleneck:
  Current: 150-250ms per asset (AssimpNet C# wrapper)
  Root cause: P/Invoke marshaling overhead + managed code mesh combination
  Solution: Rust (direct Assimp FFI, zero-copy ops)
  Target: 80-150ms (2-3x faster)

Dependency Resolver Bottleneck:
  Current: 25-35ms for 50 packs (C# recursive SortedSet)
  Root cause: GC pressure + repeated lookups in deep graphs
  Solution: Go (compiled graph algorithms, process isolation)
  Target: 5-7ms (5-10x faster)

ECS Queries & Stat Modifiers:
  Current: Frame-critical but not bottleneck
  Analysis: Keep in C# (tight Unity coupling, no benefit to move)
  No change needed

MCP Server:
  Current: Python FastMCP (already optimal)
  Analysis: Dynamic dispatch perfect for 17+ tools, no benefit to move
  Keep as-is

================================================================================
ARCHITECTURE PRINCIPLES
================================================================================

1. Language Isolation
   Each language handles what it does best (Rust for native ops, Go for algos)

2. Process Boundaries
   Go runs as isolated subprocess (zero tight coupling, replaceable)

3. API Boundaries
   JSON + CLI + HTTP (language-agnostic, no cross-language exceptions)

4. Graceful Degradation
   C# fallback always works if Rust/Go unavailable

5. Transparent to Users
   Same public APIs, users unaware of language boundaries

6. Platform Agnostic
   Supports Windows, Linux, macOS with single C# codebase
