## Architecture Overview

```
┌─────────────────────────────────────────────────────────────┐
│  PackCompiler CLI (Program.cs)                              │
│  ├─ pipeline import <pack>                                  │
│  ├─ pipeline validate <pack>                                │
│  ├─ pipeline optimize <pack>                                │
│  ├─ pipeline generate <pack>                                │
│  └─ pipeline build <pack>          (orchestrates all)       │
└────────┬────────────────────────────────────────────────────┘
         │
         ├─→ AssetImportService     (GLB/FBX → ImportedAsset)
         ├─→ AssetValidationService (config + asset validation)
         ├─→ AssetOptimizationService (mesh decimation LOD)
         ├─→ PrefabGenerationService (.prefab YAML output)
         ├─→ AddressablesService     (catalog + groups)
         └─→ DefinitionUpdateService (auto-inject refs)
            │
            ├─ Models: ImportedAsset, OptimizedAsset, Config
            └─ Tests: 7 integration tests, 100% passing
```

**Data Flow**:
```
GLB/FBX file
    ↓ [AssimpNet]
ImportedAsset (vertices, indices, normals, UVs, skeleton)
    ↓ [Validation]
Validated (polycount, scale, bounds checks)
    ↓ [Optimization]
OptimizedAsset (LOD0/1/2, screen sizes)
    ↓ [Generation]
├─ Prefab YAML (LODGroup + components)
├─ Addressables Group YAML (4 addresses)
├─ Catalog entry (metadata)
└─ Definition update (visual_asset injection)
```

## Governance

### Mandatory Asset Pipeline Workflow

All agents must follow this **8-step sequence**:

1. **Define** — Create/update `asset_pipeline.yaml`
2. **Download** — `dotnet run -- sync download <pack>`
3. **Import** — `dotnet run -- pipeline import <pack>`
4. **Validate** — `dotnet run -- pipeline validate <pack>`
5. **Optimize** — `dotnet run -- pipeline optimize <pack>`
6. **Generate** — `dotnet run -- pipeline generate <pack>`
7. **Verify** — `dotnet run -- pipeline build <pack>`
8. **Commit** — Git commit all artifacts + updated definitions

**MUST NOT**:
- Manually edit game definitions when assets change
- Skip validation/optimization steps
- Create ad-hoc asset directories
- Hardcode polycount targets in C#
- Use separate/legacy tools

**Updated in**: `CLAUDE.md` (Asset Pipeline Governance section)

**Impact**: LOD quality may be suboptimal for complex meshes

**Severity**: Medium
**Impact**: HTML files hang instead of failing gracefully
**Workaround**: Add file type validation at import start
**Fix timeline**: Week 4 (add magic byte checks)
