## v0.11.0 Known Regressions (Being Fixed)

### Issue 1: F9/F10 Key Input Broken (RT-003, RT-004)

**Symptom**: F9 and F10 key presses do not toggle debug overlay or mod menu.

**Root Cause**: Win32 global key watcher hook fires and posts messages, but `KeyInputSystem.OnInput` callback is never invoked. The message dispatch chain is broken or the system is not registered properly.

**Impact**:
- Debug overlay (F9) cannot be opened
- Mod menu overlay (F10) cannot be toggled
- All overlay automation tests fail (OVL-001/002)
- Native "Mods" button cannot function as F10 equivalent (NATIVE-002)

**Fix Status**: ЁЯФД IN_PROGRESS
**Target**: v0.12.0
**Investigation**: Check `KeyInputSystem.OnEnable()` watcher registration; verify Win32 message loop integration in `RuntimeDriver.Update()`.

---

### Issue 2: RuntimeDriver.OnDestroy Fires Early (RT-005, NATIVE-001/003)

**Symptom**: Root GameObject with `HideAndDontSave` flag is destroyed at frame ~6 (approximately 100ms) instead of persisting тЙе 600 frames (~10s).

**Root Cause**: Unknown. Suspected interactions:
- Scene unload lifecycle triggering premature cleanup
- Conflict with DINO's two-boot cycle destroying the root
- `OnDestroy` being called unexpectedly from `RuntimeDriver` or BepInEx lifecycle

**Impact**:
- `HideAndDontSave` persistence does not survive restart (RT-002 тЪая╕П PARTIAL)
- `RuntimeDriver.Update()` loop terminates early, stopping KeyInputSystem polling
- Native "Mods" button cannot persist in main menu (NATIVE-001/003)
- F9/F10 fixes (Issue 1) also blocked until RuntimeDriver survives тЙе 600 frames

**Fix Status**: ЁЯФД IN_PROGRESS
**Target**: v0.12.0
**Investigation**: Check `RuntimeDriver.OnDestroy()` stack trace; verify `HideAndDontSave` flag is set before first scene load; test persistence across explicit `SceneManager.LoadScene()` calls.

---

### Issue 3: UGUI Overlay Alpha Visible by Default (OVL-006)

**Symptom**: Mod menu overlay is partially visible (alpha &gt; 0) before user presses F10.

**Root Cause**: `HudStrip.AlphaBase` suppression logic not triggering until first F10 press.

**Impact**: Visual clutter on screen until user explicitly opens overlay.

**Fix Status**: тЪая╕П PARTIAL
**Target**: v0.12.0 (verify in live game)
**Investigation**: Merged fix in v0.11.0; requires live game QA to confirm `AlphaBase` is 0 until F10 pressed.
