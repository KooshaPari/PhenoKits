# ADR-001: OpenAPI-Driven API Synchronization

## Status
Accepted

## Context
API synchronization requires:
1. Contract-first development
2. Version management
3. Consumer-provider alignment
4. Automated validation

## Decision
Use OpenAPI Specification 3.1 as the canonical API contract:
1. API definitions in OpenAPI YAML/JSON
2. Code generation from specifications
3. Contract testing with schemathesis
4. Version-controlled specifications

## Implementation

```yaml
openapi: 3.1.0
info:
  title: Phenotype API
  version: 1.0.0
paths:
  /features:
    get:
      operationId: listFeatures
      responses:
        '200':
          description: Feature list
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/FeatureList'
```
