# ADR-002: Contract Testing Strategy

## Status
Accepted

## Context
API contracts need validation:
1. Provider implementation matches spec
2. Consumer expectations are met
3. Breaking changes are detected
4. Compatibility is verified

## Decision
Implement multi-layer contract testing:
1. Schema validation (openapi-spec-validator)
2. Provider tests (schemathesis)
3. Consumer-driven contract tests (Pact)
4. Breaking change detection (oasdiff)

## Implementation

```yaml
contract-tests:
  steps:
    - oasdiff breaking base.yaml head.yaml
    - schemathesis run --spec api.yaml
    - pact verify --provider app
```
