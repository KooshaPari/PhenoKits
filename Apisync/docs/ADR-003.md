# ADR-003: API Version Management

## Status
Accepted

## Context
API evolution requires:
1. Backward compatibility
2. Version lifecycle management
3. Deprecation communication
4. Migration paths

## Decision
Implement semantic versioning for APIs:
1. URL path versioning (/v1/, /v2/)
2. Sunset headers for deprecation
3. Minimum 6-month deprecation period
4. Migration documentation requirements

## Implementation

```go
func VersionMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        version := c.Param("version")
        c.Set("api_version", version)
        c.Next()
    }
}

// Deprecation headers
w.Header().Set("Deprecation", "true")
w.Header().Set("Sunset", "Sat, 01 Jun 2024 00:00:00 GMT")
```
