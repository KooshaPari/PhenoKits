# ADR-002: Plugin API Design

## Status
**Accepted**

## Context
Plugin API must be stable, versioned, and comprehensive.

## Decision
- Interface definition language (IDL)
- Semantic versioning for API
- Backward compatibility guarantees
- Plugin capability negotiation
- Host services for plugins
- Event subscription model

## Consequences
- Stable plugin ecosystem
- Evolution constraints
- Good developer experience
