# Phenotype Org Worklog System

Worklogs are the canonical audit trail for research, decisions, and significant findings across the Phenotype organization. They are organized by category and queryable via the **[INDEX](./INDEX.md)**.

## Quick Start

| Purpose | Go To |
|---------|-------|
| **Find recent work** | [INDEX.md](./INDEX.md) — sorted by date (most recent first) |
| **Browse by category** | [INDEX.md](./INDEX.md) **By Category** section |
| **Find by repo** | [INDEX.md](./INDEX.md) **All Worklogs** table |
| **Write a new entry** | See below: [Writing Worklogs](#writing-worklogs) |
| **Regenerate INDEX** | `tooling/worklog-aggregator/target/release/worklog-aggregator` |

## Per-Repo Worklogs

Each repository may have its own worklog(s):
- **Root-level**: `<repo>/worklog.md` — general project work (often a template)
- **Organized**: `<repo>/worklogs/*.md` — category-specific docs (rarely used)

## Writing Worklogs

Write an entry when:
- **Research completes** — findings, decisions, recommendations
- **Design decision made** — rationale, tradeoffs, ADR
- **Issue found** — duplication, performance, security, compliance
- **Work completes** — phase closure, feature ship, refactor
- **Planning** — fork candidates, migration plans, roadmaps

### Format

```markdown
## [YYYY-MM-DD] Title — Quick Category Hint

**Project tags:** `[ProjectName]` or `[cross-repo]`

**Finding/Decision:**
- What did you discover or decide?
- Why does it matter?

**Impact:**
- Downstream implications
- Next steps

**References:**
- Spec/PR/Issue links
```

### Guidelines

1. **Date** — Every entry must have ISO format date (`YYYY-MM-DD`)
2. **Category** — Tag with primary category (ARCHITECTURE, DEPENDENCIES, etc.)
3. **Scope** — Project tag or `[cross-repo]`
4. **Queryability** — Use consistent headers for searchability
5. **Location** — Always write to `worklogs/` (never root-level docs)

## Regenerating the INDEX

The INDEX is auto-generated by the **worklog-aggregator** Rust tool:

```bash
cd /Users/kooshapari/CodeProjects/Phenotype/repos/tooling/worklog-aggregator
cargo build --release
./target/release/worklog-aggregator
```

**What it does:**
- Walks all repos (excludes `.archive/`, `.worktrees/`)
- Collects `worklog.md` (any depth ≤4) and `worklogs/*.md` files
- Extracts dates and categories
- Writes `worklogs/INDEX.md` sorted by most-recent first
- Flags each entry as **Active** (content) or **Template** (placeholder)

## Quick Queries

### Find by date
Open `INDEX.md` **All Worklogs** → sort by `Date` column

### Find by category
Open `INDEX.md` **By Category** section

### Search within category
```bash
# All ARCHITECTURE entries
grep -l "ARCHITECTURE" worklogs/*.md

# Search for a topic across all worklogs
grep -r "hexagonal\|polyrepo" worklogs/
```

### Find by repo
Open `INDEX.md` **All Worklogs** → filter by `Repo` column
